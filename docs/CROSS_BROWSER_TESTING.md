# 跨浏览器兼容性测试指南

## 概述

本文档提供了 Memo Web Frontend 的跨浏览器兼容性测试计划和手动测试清单。该应用使用原生 HTML、CSS 和 JavaScript 构建，目标是在所有主流浏览器和移动设备上提供一致的用户体验。

## 目标浏览器

### 桌面浏览器
- **Chrome**: 最新 2 个版本
- **Firefox**: 最新 2 个版本
- **Safari**: 最新 2 个版本（macOS）
- **Edge**: 最新 2 个版本

### 移动浏览器
- **Mobile Safari**: iOS 12+
- **Chrome Mobile**: 最新 2 个版本（Android）
- **Samsung Internet**: 最新版本
- **Firefox Mobile**: 最新版本

## 必需的浏览器特性

应用依赖以下现代浏览器特性：

- ES6+ (classes, modules, async/await, arrow functions)
- Fetch API
- Promise
- CSS Grid 和 Flexbox
- CSS Custom Properties (CSS Variables)
- DOM API (querySelector, addEventListener, etc.)

## 测试环境设置

### 桌面测试
1. 在本地或测试服务器上运行应用
2. 确保后端 API 服务正在运行
3. 在每个目标浏览器中打开应用
4. 使用浏览器开发者工具检查控制台错误

### 移动设备测试
1. **真实设备测试**（推荐）：
   - 在同一网络上访问应用
   - 使用设备的 IP 地址（例如：http://192.168.1.100:3000）
   
2. **浏览器开发者工具模拟**：
   - Chrome DevTools: Device Mode (F12 → Toggle Device Toolbar)
   - Firefox: Responsive Design Mode (Ctrl+Shift+M)
   - Safari: Responsive Design Mode (Develop → Enter Responsive Design Mode)

## 核心功能测试清单

### 1. 页面加载和初始渲染

#### 测试步骤
1. 打开应用 URL
2. 观察页面加载过程

#### 验证点
- [ ] 页面在 2 秒内完成加载
- [ ] 标题 "Memo" 正确显示
- [ ] 任务创建表单正确渲染
- [ ] 任务列表容器正确显示
- [ ] 如果有任务，任务列表正确加载并显示
- [ ] 如果没有任务，显示空状态提示信息
- [ ] 没有 JavaScript 控制台错误
- [ ] 没有 CSS 样式问题（布局正常）

#### 浏览器特定注意事项
- **Safari**: 检查 CSS Grid 和 Flexbox 布局
- **Edge**: 验证 CSS 自定义属性正确应用
- **Firefox**: 检查字体渲染和间距

---

### 2. 创建新任务

#### 测试步骤
1. 在标题输入框中输入任务标题
2. 在描述输入框中输入任务描述
3. 点击"添加"按钮

#### 验证点
- [ ] 输入框正确接收输入
- [ ] 实时验证正常工作（空标题显示错误）
- [ ] 提交按钮在标题为空时禁用
- [ ] 点击提交后显示加载状态
- [ ] 任务创建成功后显示成功通知
- [ ] 新任务出现在任务列表中
- [ ] 表单输入框被清空
- [ ] 焦点返回到标题输入框

#### 浏览器特定注意事项
- **Mobile Safari**: 测试虚拟键盘行为，确保输入框不被遮挡
- **Chrome Mobile**: 验证自动填充不干扰表单
- **所有浏览器**: 测试 Enter 键提交表单

---

### 3. 切换任务完成状态

#### 测试步骤
1. 点击未完成任务的复选框
2. 观察任务状态变化
3. 再次点击复选框切换回未完成状态

#### 验证点
- [ ] 复选框立即响应点击
- [ ] 任务文本应用删除线样式（已完成）
- [ ] 任务背景颜色变化（已完成状态）
- [ ] 任务在已完成和未完成区域之间移动
- [ ] 状态切换有平滑的过渡动画
- [ ] 如果 API 失败，状态回滚并显示错误

#### 浏览器特定注意事项
- **Safari**: 验证 CSS 过渡动画正常工作
- **Firefox**: 检查复选框样式和自定义外观
- **移动浏览器**: 确保触摸目标足够大（至少 44x44px）

---

### 4. 删除任务

#### 测试步骤
1. 将鼠标悬停在任务项上（桌面）或直接查看（移动）
2. 点击删除按钮（×）
3. 观察任务被删除

#### 验证点
- [ ] 删除按钮在悬停时显示（桌面）或始终可见（移动）
- [ ] 点击删除按钮后任务立即从列表中移除
- [ ] 删除动画平滑（淡出效果）
- [ ] 如果 API 失败，任务重新出现并显示错误
- [ ] 删除成功后显示成功通知

#### 浏览器特定注意事项
- **移动浏览器**: 删除按钮应始终可见，不依赖悬停
- **所有浏览器**: 验证触摸/点击事件正确处理

---

### 5. 错误处理

#### 测试步骤
1. 停止后端服务器
2. 尝试创建、更新或删除任务
3. 观察错误处理

#### 验证点
- [ ] 显示用户友好的错误消息
- [ ] 错误通知清晰可见
- [ ] 错误消息自动消失或可手动关闭
- [ ] 应用不崩溃或进入不可用状态
- [ ] 可以重试失败的操作

#### 测试场景
- 网络错误（服务器不可用）
- 超时错误（模拟慢速网络）
- 验证错误（提交空标题）
- 服务器错误（500 响应）

---

### 6. 响应式设计

#### 测试步骤
1. 在不同屏幕尺寸下测试应用
2. 旋转移动设备（纵向/横向）
3. 调整浏览器窗口大小

#### 验证点

**桌面视图（≥1024px）**
- [ ] 任务列表使用多列布局（如适用）
- [ ] 表单元素水平排列
- [ ] 悬停效果正常工作
- [ ] 最大内容宽度限制生效

**平板视图（640px-1023px）**
- [ ] 布局适当调整
- [ ] 所有内容可见且可访问
- [ ] 触摸目标足够大

**移动视图（<640px）**
- [ ] 单列布局
- [ ] 表单元素垂直堆叠
- [ ] 删除按钮始终可见
- [ ] 文本大小适合阅读
- [ ] 触摸目标至少 44x44px
- [ ] 没有水平滚动

#### 浏览器特定注意事项
- **Mobile Safari**: 测试地址栏隐藏/显示时的布局变化
- **Chrome Mobile**: 验证视口单位（vh, vw）正确工作
- **所有移动浏览器**: 测试缩放功能

---

### 7. 键盘导航和可访问性

#### 测试步骤
1. 仅使用键盘导航应用
2. 使用屏幕阅读器测试（如可用）

#### 验证点
- [ ] Tab 键可以导航到所有交互元素
- [ ] 焦点指示器清晰可见
- [ ] Enter 键可以提交表单
- [ ] Space 键可以切换复选框
- [ ] Escape 键可以关闭通知
- [ ] 焦点顺序符合逻辑
- [ ] 所有交互元素有适当的 ARIA 标签
- [ ] 屏幕阅读器可以读取所有内容

#### 浏览器特定注意事项
- **Safari + VoiceOver**: 测试 macOS 和 iOS 上的屏幕阅读器
- **Chrome + NVDA/JAWS**: 测试 Windows 上的屏幕阅读器
- **Firefox**: 验证焦点管理

---

### 8. 性能测试

#### 测试步骤
1. 使用浏览器开发者工具的性能分析器
2. 测量页面加载时间
3. 测量交互响应时间

#### 验证点
- [ ] 初始页面加载 < 2 秒
- [ ] 用户交互后 < 100ms 提供视觉反馈
- [ ] 任务列表渲染流畅（即使有大量任务）
- [ ] 没有明显的卡顿或延迟
- [ ] 内存使用合理（无内存泄漏）

#### 测试工具
- Chrome DevTools: Lighthouse, Performance tab
- Firefox DevTools: Performance tab
- Safari Web Inspector: Timelines

---

### 9. 视觉一致性

#### 测试步骤
1. 在所有目标浏览器中打开应用
2. 比较视觉呈现

#### 验证点
- [ ] 颜色一致（CSS 变量正确应用）
- [ ] 字体渲染一致
- [ ] 间距和布局一致
- [ ] 按钮和表单元素样式一致
- [ ] 图标和符号正确显示
- [ ] 过渡和动画效果一致

#### 已知差异
- 字体渲染在不同操作系统上可能略有不同（可接受）
- 滚动条样式因浏览器而异（可接受）
- 表单元素的默认样式可能不同（应通过 CSS 规范化）

---

### 10. 边界情况和压力测试

#### 测试场景

**长文本**
- [ ] 非常长的任务标题（接近 200 字符限制）
- [ ] 非常长的任务描述（接近 1000 字符限制）
- [ ] 文本正确换行，不溢出容器

**大量任务**
- [ ] 创建 50+ 个任务
- [ ] 列表渲染性能良好
- [ ] 滚动流畅
- [ ] 操作响应及时

**特殊字符**
- [ ] 任务标题包含特殊字符（<, >, &, ", '）
- [ ] 正确转义，防止 XSS
- [ ] 表情符号正确显示

**网络条件**
- [ ] 慢速网络（3G）下的行为
- [ ] 间歇性网络连接
- [ ] 离线状态（显示适当错误）

## 已知问题和解决方案

### Safari 特定问题

**问题**: CSS Grid 在旧版 Safari 中可能有兼容性问题
**解决方案**: 使用 Flexbox 作为后备布局

**问题**: 日期格式化可能不一致
**解决方案**: 使用 `Intl.DateTimeFormat` 或自定义格式化函数

### Mobile Safari 特定问题

**问题**: 虚拟键盘可能遮挡输入框
**解决方案**: 使用 `scrollIntoView()` 确保输入框可见

**问题**: 100vh 包含地址栏高度
**解决方案**: 使用 JavaScript 计算实际视口高度或使用 CSS `dvh` 单位（如支持）

### Firefox 特定问题

**问题**: 自定义复选框样式可能不一致
**解决方案**: 使用 `-moz-appearance: none` 重置默认样式

### Edge 特定问题

**问题**: 旧版 Edge（EdgeHTML）不支持某些 ES6 特性
**解决方案**: 仅支持基于 Chromium 的 Edge（最新 2 个版本）

## 测试频率

### 开发阶段
- 每个主要功能完成后进行快速测试
- 每周进行一次完整的跨浏览器测试

### 发布前
- 完整的跨浏览器测试（所有目标浏览器）
- 真实设备测试（至少 iOS 和 Android 各一台）
- 性能和可访问性审计

### 发布后
- 监控用户报告的浏览器特定问题
- 定期回归测试（每月或每次更新后）
